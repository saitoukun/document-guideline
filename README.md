## Best Practices <a name = "best_practices"></a>

### 心がけ

- 叙述形式で伝わるような文章を書く
- 読み手の認知コストを最小に抑えることを意識する
- プレーンテキストで再利用しやすい状態にする

### 技術者向け

- Markdownを使用して構造化する
- Table of Contentsを用意する
- git管理で差分を残す
